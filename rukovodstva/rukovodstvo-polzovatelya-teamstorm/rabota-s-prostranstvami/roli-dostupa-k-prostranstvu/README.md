# Роли доступа к пространству

В каждом пространстве Системы реализована ролевая модель доступа со следующими  встроенными ролями:

1. Администратор пространства со следующими правами по управлению пространством:
   * редактирование имени и описания пространства;
   * управление доступами;
   * &#x20;управление интеграциями;
   * &#x20;удаление пространства;
   * &#x20;управление типами;
   * &#x20;управление атрибутами;
   * &#x20;управление процессами:
2. Пользователь пространства со следующими правами на просмотр и редактирование контента пространства:
   * работа с папками;
   * работа с представлениями;
   * работа с задачами;
   * работа с расширениями;
   * выгрузка данных;
3. Гость пространства с правами только на просмотр контента.

{% hint style="info" %}
Администратор пространства по умолчанию также обладает всеми правами пользователя и правами на просмотр контента пространства
{% endhint %}
